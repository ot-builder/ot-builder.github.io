Namespace FontIo

IO Procedures

FontIo.readFont<GS extends Data.Order<Ot.Glyph>>(sfnt,​gsf,​config?):​Ot.Font<GS>
· sfnt:Ot.Sfnt SFNT to parse
· gsf:Data.OrderStoreFactoryWithDefault<Ot.Glyph,​GS> Factory to create a glyph store
· config?:FontIo.FontIoConfig IO configuration

Creates a Ot.Font form a Ot.Sfnt. The paraemter gsf is used to provide the glyph store factory to create the glyph store. When using with Ot.ListGlyphStore, Ot.ListGlyphStoreFactory is recommended to be provided for gsf parameter.

FontIo.writeFont<GS extends Data.Order<Ot.Glyph>>(font,​config?):​Ot.Sfnt
· font:Ot.Font<GS> The font to write
· config?:FontIo.FontIoConfig IO configuration

Creates a Ot.Sfnt form a Ot.Font.

IO Configuration

Datatype FontIo.FontIoConfig

Members

  • glyphStore?:
    • statOs2XAvgCharWidth?:​boolean

      Whether to recalculate xAvgCharWidth property in OS/2 table when writing fonts. Defaults to true.

  • cff?:
    • doLocalOptimization?:​boolean

      Whether to perform local optimization when writing the CFF/CFF2 table. Defaults to true.

    • doGlobalOptimization?:​boolean

      Whether to perform global optimization when writing the CFF/CFF2 table. Defaults to true.

  • ttf?:
    • gvarOptimizeTolerance?:​boolean

      The error tolerance when writing the gvar table. Defaults to 1/128. With higher tolerance the gvar table writer would do more aggressive IUP optimization.

  • encoding?:
    • statOs2UnicodeRanges?:​boolean

      Whether to recalculate ulUnicodeRange1 ... ulUnicodeRange4 property in OS/2 table when writing fonts. Defaults to true.

  • glyphNaming?:
    • namer?:​Ot.GlyphNamer

      The glyph namer. Defaults to Ot.StandardOtGlyphNamer.
